What is Sass?

Sass stands for Syntactically Awesome Stylesheets.

Sass is a CSS preprocessor.

Preprocessors extend CSS by adding special features like variables, nested rules, and mixins to regular CSS.

Makes coding simpler and more efficient.


Installing Sass

Installing on a Mac OS C (Homebrew): 

brew install sass/sass/sass
Installing anywhere (Node.js): 
npm install -g sass
Installing on Windows (Chocolatey): 
choco install sass
After you finished installing Sass on the command line, run sass --version to make sure it is installed properly. It should include 1.25.3


Sass vs. SCSS

Sass can be written in two different ways, with SASS syntax and with SCSS syntax. Overall, they are quite similar, but SASS uses more whitespace, while SCSS uses brackets and resembles CSS


Why Use Sass? (Simple example)

Let’s say you want to create a website with these three main colors:

#333333
#FFC0CB
#BADA55

Instead of typing these HEX values a bunch of times, you can use Sass and store them as variables.


Variables

Variables are a way to store information that can be re-used later.

Variables can store all the types supported by CSS (numbers, strings, colors) and other SASS-specific values (booleans, null, maps, function references).

Variable declaration syntax: $<variable>:<value>

SCSS:

CSS:

	
      // declaring the variables
      $primary1: #333333;
      $primary2: #FFC0CB;
      $primary3: #BADA55;
      $fontsize: 18px;

      // using the variables
      header{
        color: $primary2;
      }
      body{
        color: $primary1;
        font-size: $fontsize;
      }
      
	
	header{
          color: #FFC0CB;
	}
	body{
	  color: #333333;
	}

Nesting

Nesting is a shortcut to group selectors.

Instead of writing many lines for an element, you enclose a selector inanother selector.

Many CSS properties have the same prefix, and with Sass you can write them as nested properties.

SCSS:

CSS:

      
      body{
        color: $primary1;
        font: {
          family: Helvetica, sans-serif;
          size: $fontsize;
          weight: bold;
        };
        text: {
          align: center;
          transform: lowercase;
          overflow: hidden;
        }
      }
    
      
      body {
        color: #333333;
        font-family: Helvetica, sans-serif;
        font-size: 18px;
        font-weight: bold;
        text-align: center;
        text-transform: lowercase;
        text-overflow: hidden;
      }

Import

The @import directive allows you to include the content of one file in another (similar to the CSS @import directive).

More efficient than the CSS @import because in CSS, an extra HTTP request is created each time its called but in Sass no extra call is required at runtime. 


	// foundation/_code.scss
	code{
	  padding: .25em;
	  line-height: 0;
	}
	
	
	// style.scss
	@import 'foundation/code.scss'
All the code in the code.scss file in the foundation repo is automatically included in style.scss now.


Mixins

The @mixin directive lets you create a block of code that groups CSS declarations to be reused throughout the website. 

	
	@mixin default{
    	  font-size: $fontsize;
	  width: 100%;
	  padding: 10px;
	  color: $primary3;
	}

The @include directive is used to include a mixin in a selector. 

	p {
	  @include default;
	}
The above code compiles to: 
	p {
	  font-size: 18px;
	  width: 100%;
	  padding: 10px;
	  color: #BADA55;
	}

Mixins accept arguments, so you can pass variables to a mixin. 

	@mixin colored( $color){
	  color: $color;
	}

	p {
	  @include colored( $primary2);
	}


Extend

Syntax: @extend <selector>

Tells Sass that one selector inherits the styles of another.

The @extend directive is useful if you have almost identically styled elements that only differ in some small details.

This is different from the @include directive in mixins because @extend updates style rules that contain the selector to contain the extending selectorf but mixins copy styles into the current style rule.

SCSS:

CSS:

	
	.first{
	  background-color: $primary3;
	}
	
	.second{
	  @extend .firstp;
	  font-style: italic;
	}
      
	
	.first, .second {
	  background-color: #FFC0CB;
	}
	
	.second {
	  background-color: #ff748c;
	}

Comparison and arithmetic operators

Operators like +, -, *, /, >, <=, etc. perform standard operations in Sass.

The addition operator is cool because it can also be used to concatenate strings.

Division also works for variables.

Note: It only works for numbers with compatible numbers. 

	
	h2{
	  font-size: 5px + 2em; // error incompatible units
	
	  font-size: 5px + 2; // 7px
	}


Functions

SASS allows for functions using the @function call.

Functions can take parameters and output results using @return. 

      
      @function pow( $base, $exponent){
        $result: 1;
        @for $_ from 1 through $exponent{
          $result: $result * $base;
        }
        @return $result;
      }

      main{
        font-size: pow( 3,3) * 1px; // font-size: 8px
      }

Sass contains many built-in modules also, like color modules, list, map, math, selector, and string.

      
      .firstp{
        background-color: $primary2;
      }

      .secondp{
        background-color: darken( $primary2, 15%);
      }
